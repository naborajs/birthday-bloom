## 📋 How to Use These Configurations

### For Local Testing:

1. Copy the configuration block you want
2. Create/edit `.env.local` in the project root:
   ```bash
   # Windows (PowerShell)
   notepad .env.local
   
   # Mac/Linux
   nano .env.local
   ```
3. Paste the configuration
4. Save and restart your dev server:
   ```bash
   npm run dev
   ```
5. The site will automatically pick up the new values

### For Vercel Deployment:

1. Go to https://vercel.com/dashboard
2. Select your project
3. Go to Settings → Environment Variables
4. For each line in the configuration:
   - Name: Variable name (e.g., `VITE_BIRTHDAY_NAME`)
   - Value: Variable value (e.g., `Sarah`)
5. Click "Save"
6. Redeploy by pushing to GitHub or clicking "Redeploy"

### For Other Platforms:

**Firebase:** Firebase Console → Project Settings → Environment Variables
**Netlify:** Site settings → Build & deploy → Environment
**AWS Amplify:** App settings → Environment variables
**Docker:** Pass as `-e VAR=VALUE` flags

## 💡 Pro Tips

1. **Test Before Deploying:**
   ```bash
   npm run dev  # Test locally
   npm run build && npm run preview  # Test build output
   ```

2. **Use Existing Photos:**
   - Don't have photos? Use Unsplash URLs above
   - They're free and unlimited
   - Change the `?search=` parameter for different themes

3. **Customize the Message:**
   - Add emojis 😊 🎉 💕
   - Include their name
   - Be personal and sincere
   - Multiple lines work!

4. **Mobile Friendly:**
   - Test on your phone after deploying
   - All configs work on mobile
   - Adjust `VITE_ANIMATION_INTENSITY` if it lags

5. **Older Users:**
   - Use `VITE_REDUCED_MOTION=true`
   - Set `VITE_TEXT_SIZE=large`
   - Lower `VITE_ANIMATION_INTENSITY=low`
